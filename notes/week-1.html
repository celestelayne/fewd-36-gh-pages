JSON

  • Your preferences for Sublime are one big JSON object.
  • JSON stands for JavaScript Object Notation.
  • It's a way to package up data. A JSON package can then be sent/received by many different applications or API's. It's the most popular data exchange format on the internet today.
  • JSON was popularized by Douglas Crockford. Before that, we were stuck with the more verbose XML for data exchange.
  • JSON object starts and ends with a curly braces.
    { }
  • It is a notation for desribing an object.
  • What's an object? An object is a list of properties.
    { chips, cookies, popcorn }
  • What's a property? A property has a name and a value
  • A property's name/value is separated by a colon.
    { chips: "tortilla" }
  • Just like a list in a sentance, properties are separated by comma's.
    { chips: "tortilla", cookies: "chocolate chip", popcorn: "kettle corn" }
  • A property name must always be a string. The value could be any type of object: string, number, boolean, an array, or another object (object inception anyone?). We will talk more about this once we get into JavaScript.

Pro Tips

1. Never have to remember to save an html file again! Setup Sublime with: 

save_on_focus_lost

{ "save_on_focus_lost": true }

2. Always open a project in Sublime in "folder mode". You do this by dragging the folder from the Finder onto the Sublime icon in the dock. This way you see all your source files/folders for that project in the left sidebar in Sublime.
